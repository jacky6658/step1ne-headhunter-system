# AI 提示詞使用指南

本系統在人選卡片中提供兩組 AI 分析提示詞，分別對應不同的工作階段與分析目的。

---

## 一、AI 總結 提示詞

### 位置
人選卡片 → `AI 總結` 分頁 → 「複製 AI 分析提示詞」按鈕

### 目的
**廣度分析** — 快速了解人選的全貌，找出他最適合的職缺方向。

### 使用時機
| 情境 | 說明 |
|------|------|
| 剛接觸新人選 | 從 LinkedIn / 104 / 爬蟲匯入一位新候選人，需要快速掌握他的背景 |
| 人選篩選階段 | 手上有多位候選人，需要快速判斷每個人的市場定位 |
| 還不確定要推什麼職缺 | 想讓 AI 幫忙從系統所有職缺中找出最匹配的 Top 3 |
| 內部會議討論 | 需要一份人選的快速摘要給團隊看 |

### 分析內容
1. **一句話定位** — 精準描述候選人核心價值
2. **Top 3 最匹配職缺** — 從系統所有職缺中挑出最適合的 3 個（含匹配原因、落差、評分）
3. **能力分析** — 核心優勢 + 能力缺口
4. **風險評估** — 穩定性、薪資、到職時間等風險
5. **建議顧問提問** — 5 題分四類（技能驗證/行為面試/動機探索/風險確認）
6. **顧問行動建議** — 下一步該做什麼

### 資料來源
- 呼叫 `GET /api/jobs` 取得系統**所有職缺**
- 人選卡片上的所有基本資料 + 工作經歷

### 智慧補全邏輯
- 基本資料「未填」時，AI 會從工作經歷交叉推斷，並標註「根據工作經歷推斷」
- 技能欄空白時，自動附帶工作描述內容供 AI 提取技能

## 二、AI 匹配結語 提示詞

### 位置
人選卡片 → `AI 匹配結語` 分頁 → 「複製 AI 匹配分析提示詞」按鈕

### 目的
**深度匹配** — 針對人選的「目標職缺」進行一對一深度分析，產出可直接用於客戶溝通的結果。

### 使用時機
| 情境 | 說明 |
|------|------|
| 決定推薦某職缺 | 已確認要把人選推薦到特定職缺，需要深度評估匹配度 |
| 準備推薦報告 | 需要產出推薦話術，準備向用人主管 Pitch |
| 面試前準備 | 需要針對性面試問題，確認人選是否真的適合 |
| 薪資談判前 | 需要分析人選期望 vs 職缺預算，擬定談判策略 |
| 風險評估 | 人選有疑點（頻繁跳槽、技能落差），需要更細緻的風險分析 |

### 分析內容
1. **匹配度總評（0-100 分）** — 含各維度評分：技能、經驗、薪資、文化適配、穩定性
2. **核心優勢（Strengths）** — 3-5 項相對目標職缺的核心競爭力
3. **風險與劣勢（Risks & Gaps）** — 區分「硬傷」vs「可補足」
4. **薪資分析** — 期望 vs 預算匹配 + 談判策略建議
5. **建議顧問提問** — 5-8 題分四類（技能驗證/行為面試/動機探索/風險確認）
6. **推薦話術（Pitch to Client）** — 3-5 句可直接用於向客戶推薦的話術
7. **顧問行動建議（Next Steps）** — 具體下一步方案

### 資料來源
- 呼叫 `GET /api/jobs/{targetJobId}` 取得**單一目標職缺**詳細資訊
- 人選卡片上的所有基本資料 + 工作經歷

### 前置條件
- 人選需要已指定「目標職缺」（若未指定，提示詞會標註並根據職缺名稱推斷）

---

## 三、一句話總結

> **AI 總結** = 「這個人是誰？適合什麼工作？」（選才階段）
>
> **AI 匹配結語** = 「這個人適不適合這個職缺？要怎麼推薦給客戶？」（推薦階段）

## 四、兩者對比

| 比較項目 | AI 總結 | AI 匹配結語 |
|---------|---------|------------|
| 分析方向 | 廣度 — 人選適合什麼？ | 深度 — 人選適不適合這個職缺？ |
| 職缺範圍 | 系統所有職缺 | 僅目標職缺 |
| 匹配分析 | Top 3 快速匹配 | 單一職缺多維度深度匹配 |
| 推薦話術 | 無 | 有（Pitch to Client） |
| 薪資談判 | 僅風險提醒 | 含談判策略建議 |
| 面試問題 | 5 題通用 | 5-8 題針對性 |
| 工作階段 | 選才篩選階段 | 推薦決策階段 |

## 五、建議使用流程

```
新人選進入系統
    │
    ▼
┌─────────────┐
│  AI 總結     │  ← 快速掌握人選全貌，找出適合的職缺方向
│  (廣度分析)   │
└──────┬──────┘
       │ 確認要推薦某個職缺
       ▼
┌─────────────┐
│ 指定目標職缺  │  ← 在人選卡片上綁定目標職缺
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ AI 匹配結語   │  ← 深度評估 + 產出推薦報告 + 面試準備
│ (深度匹配)    │
└──────┬──────┘
       │
       ▼
  向客戶推薦 / 安排面試
```

---

## 六、注意事項

1. **AI 不會編造資訊** — 提示詞已加入嚴格約束，AI 會區分「已確認資料」vs「從工作經歷推斷的資料」
2. **工作經歷是核心** — 即使基本資料未填，只要工作經歷完整，AI 仍能進行有效分析
3. **需要外部 AI 工具** — 複製提示詞後需貼到 ChatGPT / Claude 等 AI 工具使用
4. **目標職缺需先設定** — AI 匹配結語的提示詞會自動帶入目標職缺 API，若未設定則以職缺名稱推斷
