## 角色權限差異

### ADMIN（管理員）

- 查看所有候選人（不受 consultant 欄位限制）
- 查看所有職缺、客戶
- 系統配置管理
- 用戶管理（CRUD）
- 系統日誌查看

### REVIEWER（獵頭顧問）

- 只能查看自己負責的候選人（依 `consultant` 欄位篩選）
- 可新增候選人（自動指派為自己負責）
- 可更新自己負責的候選人狀態
- 可查看所有職缺與客戶（唯讀）

## 龍蝦識別用戶的方式

當用戶與龍蝦互動時，龍蝦應：

1. **確認用戶身份**：詢問或根據上下文判斷當前用戶是誰（Admin / Jacky / Phoebe）
2. **對應 consultant 欄位**：
   - Jacky → `consultant: "Jacky"`
   - Phoebe → `consultant: "Phoebe"`
   - Admin → 不限制
3. **篩選資料範圍**：
   - REVIEWER 角色：API 查詢時加上 consultant 篩選
   - ADMIN 角色：不加篩選條件
4. **操作時自動帶入**：
   - 新增候選人時，自動填入 `recruiter` 為當前顧問名稱
   - 記錄互動時，標記操作者

### 範例

```
用戶（Jacky）：「幫我看一下我有多少待聯繫的候選人」

龍蝦流程：
1. GET /api/candidates?limit=2000
2. 篩選 consultant === "Jacky" && status === "待聯繫"
3. 回報：「Jacky 目前有 12 位待聯繫候選人」
```

## 爬蟲系統權限

| 權限 | 預設狀態 | 開啟方式 |
|------|---------|---------|
| 爬蟲操作 | **關閉** | 用戶明確說「開啟爬蟲權限」或類似指令 |

- 未授權時，龍蝦不得呼叫任何爬蟲相關 API
- 授權後，龍蝦可操作爬蟲系統的任務建立、啟動、查詢、推送等功能
- 授權範圍僅限當前對話，不跨對話保留

## 用戶管理 API

| 方法 | 端點 | 說明 |
|------|------|------|
| GET | `/api/users/all` | 所有用戶列表 |
| POST | `/api/users/login` | 用戶登入 |
| POST | `/api/users/create` | 建立新用戶 |
| PUT | `/api/users/:uid` | 更新用戶 |
| DELETE | `/api/users/:uid` | 刪除用戶 |
| GET | `/api/users/names` | 用戶名稱列表 |
